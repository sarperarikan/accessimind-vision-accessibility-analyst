1. About AccessiMind

AccessiMind is an intelligent desktop application that analyzes screen recordings of web, Android, and iOS applications for accessibility conformance. Powered by Google's Gemini AI models, it acts as a virtual accessibility expert — identifying WCAG violations, UX issues, and navigation barriers automatically.

Key Capabilities

  • AI Vision Analysis: Upload screen recordings (MP4/MOV) and receive instant, expert-level feedback on accessibility issues.
  • Multi-Platform Rules:
    • 🌐 Web: WCAG 2.1/2.2 AA Conformance analysis
    • 🤖 Android: Material Design Accessibility verification
    • 🍎 iOS: Human Interface Guidelines (HIG) accessibility checks
  • Smart Reporting: Generates detailed Jira tickets with severity levels (Critical, High, Medium, Low).
  • AI Chat: Ask follow-up questions to the AI accessibility expert about any finding.
  • Project Management: Organize analyses into projects to track progress over time.
  • Bilingual UI: Full Turkish and English interface support.

2. Getting Started

System Requirements

  • Operating System: Windows 10/11 (64-bit)
  • RAM: 4 GB minimum (8 GB recommended)
  • Disk Space: 500 MB free space
  • Internet: Required for AI analysis (Gemini API)

Installation & First Launch

1

Download the Application

Download the portable AccessiMind.exe file. No installation is required — simply run the executable.

2

Get a Gemini API Key

Visit Google AI Studio to create a free API key. This key powers the AI analysis engine.

3

Configure Settings

Navigate to the Settings tab, enter your API key, select your preferred AI model, and save.

4

You're Ready!

Upload a screen recording and start your first accessibility analysis.

4. Running an Analysis

Step-by-Step Workflow

1

Select Target Platform

Choose Web, Android, or iOS from the platform selector. Each platform loads specialized accessibility rules and expert prompts.

2

Upload Screen Recording

Drag and drop a video file (MP4, MOV) into the upload area or click to browse. The video should capture the application's user interface and interactions you want analyzed.

3

Assign to a Project (Optional)

Select an existing project from the dropdown or leave as "No Project" for standalone analysis.

4

Generate Expert Report

Click the "Generate Expert Report" button. The AI will process your video through five stages:

  1. Preparing the video
  2. Uploading to the AI engine
  3. Analyzing visual elements
  4. Generating expert insights
  5. Finalizing the Jira report
ℹ️
Tip: For best results, record interactions like navigation, form filling, button presses, and scrolling. A 30–90 second video usually provides the most actionable results.

5. Understanding Results

Each analysis generates a Jira-formatted ticket containing:

  • Title: A concise name for the finding.
  • Description: Detailed explanation of the accessibility issue.
  • WCAG Criteria: The specific WCAG success criteria violated.
  • Conformance Level: The WCAG conformance level (A, AA, AAA).
  • Observed Results: What the AI detected in the video.
  • Expected Results: The correct, accessible behavior.
  • Test Steps: Steps to reproduce and verify the issue.
  • Impact Level: Critical, High, Medium, or Low.
  • Remediation Priority: Suggested fix priority ranking.

Severity Levels

Severity levels and their meanings
Severity Meaning Example
Critical Blocks major functionality for users with disabilities Form submit button not keyboard-accessible
High Significant barrier; workaround may exist Missing alt text on informational images
Medium Moderate impact; affects user experience Insufficient color contrast ratio
Low Minor issue; best practice recommendation Missing skip navigation link

Exporting & Sharing

  • Copy as Markdown: Copy the entire Jira ticket in Markdown format to paste into Jira, Confluence, or GitHub Issues.
  • Export to Text: Download the report as a .txt file for offline review.
  • AI Chat: Ask follow-up questions about any finding by typing in the chat panel below the report.

6. Project Management

Projects help you organize accessibility analyses by application, feature, or sprint.

Creating a Project

  1. Navigate to the Projects tab.
  2. Click "New Project".
  3. Enter a project name (e.g., "E-commerce Mobile App") and an optional description.
  4. Click "Create".

Managing Projects

  • View Findings: Click on a project card to view all related analysis findings.
  • Edit Project: Use the edit button to update name or description.
  • Delete Project: Remove a project and all associated findings (with confirmation dialog).

Assigning Analyses to Projects

When running an analysis from the Accessibility Expert tab, select a project from the dropdown before starting. The results will be automatically linked to that project.

8. Keyboard Shortcuts & Accessibility

AccessiMind is designed with WCAG 2.2 AA conformance. All features are fully accessible via keyboard and screen readers.

Keyboard Navigation

Keyboard shortcuts reference
Action Shortcut
Skip to main content Tab (first element focus)
Navigate between tabs Tab / Shift+Tab
Select a tab Enter or Space
Navigate within lists/cards Arrow Keys
Close modals/dialogs Escape
Submit forms Enter

Screen Reader Support

  • All interactive elements have descriptive aria-label attributes.
  • Live regions (aria-live) announce analysis progress and status changes.
  • Semantic HTML5 landmarks (<main>, <nav>, <header>) are used throughout.
  • Focus traps are implemented for all dialogs and modals.

Visual Accessibility

  • High Contrast Mode: Toggleable from Settings for users who need enhanced contrast.
  • Focus Indicators: All interactive elements display a visible :focus-visible ring.
  • Scalable Text: All text uses rem units and scales with browser zoom settings.
  • Reduced Motion: Animations respect prefers-reduced-motion preferences.

9. FAQ & Troubleshooting

Q: What video formats are supported?

A: MP4 and MOV formats are supported. For best results, use a screen recording at 720p or higher resolution.

Q: How long should the video be?

A: Videos between 30 seconds and 2 minutes provide the best balance of coverage and analysis depth. Longer videos may take additional processing time.

Q: Where are my analyses stored?

A: All analysis history is stored locally on your machine in a data/db.json file. No data is sent to external servers except during the active AI analysis session.

Q: Can I use AccessiMind offline?

A: The application interface works offline, but an internet connection is required to run AI analyses (as it uses Google's Gemini API).

Q: The analysis failed with an error. What should I do?

A: Check the following:

  1. Verify your API key is valid and entered correctly in Settings.
  2. Ensure you have an active internet connection.
  3. Try a shorter video file (under 2 minutes).
  4. Use the "Retry" button to re-attempt the analysis.

Q: Can I use a different AI model?

A: Yes. In Settings, select a model from the dropdown or enter a custom model ID. Use "Refresh" to fetch available models from Google.

10. Privacy & Data Security

  • Local Storage: All settings, project data, and analysis history are stored locally on your machine.
  • API Key Security: Your Gemini API key is stored locally and is only sent to Google's API endpoints during analysis.
  • Video Processing: Videos are sent securely to the Gemini API for analysis and are not permanently stored by Google for training (subject to Google's Enterprise API terms).
  • No Telemetry: AccessiMind does not collect usage analytics or telemetry data.
✅
Your data stays with you. AccessiMind is designed with a privacy-first architecture. All persistent data remains on your local machine.
